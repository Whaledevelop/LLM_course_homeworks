# PDF RAG: PDF → Markdown

Локальное RAG-приложение на Streamlit. Пользователь загружает PDF-файлы, приложение преобразует их в Markdown, извлекает встроенные изображения, строит локальный векторный индекс Chroma и отвечает на вопросы только по найденным фрагментам. Для генерации ответов и embeddings используются локальные модели Ollama.

## Основные пути

### `./docs`

Документация по запуску приложения и анализу результатов его работы.

### `./data`

`documents` — загруженные PDF, созданные Markdown-файлы и извлечённые изображения. `chroma` — локальная коллекция векторной базы Chroma. Папки создаются приложением и содержат рабочие данные, поэтому исключены из Git.

### `./scripts/app.py`

Streamlit-интерфейс: загрузка PDF, запуск конвертации и индексации, очистка документов, просмотр Markdown и чат с отображением источников ответа.

### `./scripts/document_service.py`

Сохраняет загруженные файлы, запускает их конвертацию в Markdown, возвращает списки PDF и Markdown, читает Markdown для предпросмотра и очищает рабочие данные вместе с индексом.

### `./scripts/pdf_service.py`

Преобразует PDF в Markdown постранично, извлекает текстовые блоки и встроенные изображения, сохраняет изображения в папке ресурсов документа и добавляет на них Markdown-ссылки.

### `./scripts/rag_service.py`

Разбивает Markdown по заголовкам и на фрагменты, создаёт индекс Chroma, ищет релевантный контекст и передаёт его вместе с вопросом чат-модели Ollama.

### `./scripts/settings.py`

Читает параметры Ollama из `.env`, определяет пути рабочих папок и создаёт их при необходимости.

### `./requirements.txt`

Python-зависимости проекта.

### `./.env.example`

Пример локальной конфигурации URL Ollama и названий используемых моделей. Его нужно скопировать в `.env`; файл `.env` не должен попадать в Git.
