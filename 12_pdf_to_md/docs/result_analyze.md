# Анализ результатов

## Как формируется результат конвертации

После загрузки каждый PDF сохраняется в `data/documents`. Для него создаются три группы файлов:

```text
data/documents/
├── document.pdf
├── document.md
└── document_assets/
    └── images/
        ├── page-001-image-001.png
        └── ...
```

Markdown начинается с названия документа и содержит заголовок `## Страница N` для каждой страницы. Текстовые блоки и встроенные изображения упорядочиваются по вертикальной позиции на странице. Для каждого сохранённого изображения добавляется Markdown-ссылка на файл из `document_assets/images`.

Одинаковые встроенные изображения внутри одного PDF сохраняются один раз. Если формат встроенного изображения не поддерживается браузером, оно преобразуется в PNG. Число страниц и извлечённых изображений отображается в боковой панели сразу после конвертации.

Если на странице нет текстового слоя и изображений, в Markdown добавляется уведомление о необходимости OCR. Это не ошибка конвертера: приложение не выполняет распознавание сканов.

## Как формируется результат ответа

Созданные Markdown-файлы делятся по заголовкам документа и страницы, затем — на фрагменты до 1200 символов с перекрытием 200 символов. Каждый фрагмент получает embedding модели `bge-m3` и сохраняется в Chroma вместе с названием исходного Markdown-файла, заголовком страницы и идентификатором фрагмента.

При вопросе приложение извлекает из Chroma до четырёх фрагментов с наибольшей релевантностью. Эти фрагменты становятся контекстом для `qwen2.5:3b`; промпт предписывает отвечать только по этому контексту. Результат в чате состоит из:

- текста ответа;
- списка «Источники»;
- названия Markdown-файла и раздела для каждого фрагмента;
- значения релевантности и текста фрагмента в раскрывающемся блоке.

Значение релевантности позволяет сравнить источники одного запроса, но не является самостоятельной оценкой фактической точности ответа.

## Как проверять результат

1. После индексации откройте блок `Markdown preview` и проверьте, что текст страниц, заголовки и ссылки на изображения созданы для каждого загруженного PDF.
2. Откройте папку `data/documents/<имя>_assets/images` и убедитесь, что Markdown-ссылки ведут на существующие файлы изображений.
3. Задайте вопрос с ответом, явно присутствующим в документе. В источниках должен отображаться фрагмент с соответствующей страницей и текстом.
4. Задайте вопрос о факте, которого нет в загруженных документах. Ожидаемый результат — сообщение: «В загруженных документах нет информации для ответа на этот вопрос.»
5. При изменении набора PDF повторно нажмите «Сохранить и обновить индекс»: индекс Chroma пересоздаётся целиком, поэтому результаты не должны опираться на удалённые документы.

## Интерпретация

Качество результата нужно оценивать на двух уровнях. Сначала проверяется конвертация: полнота текстового слоя, корректность порядка блоков и наличие изображений. Затем проверяется RAG-ответ: найден ли релевантный источник и соответствует ли ответ тексту этого источника.

Если ответ неверен, сначала проверьте Markdown и его источник в интерфейсе. Если Markdown корректен, но нужный фрагмент не попал в источники, причиной могут быть параметры разбиения, embedding-модель или формулировка вопроса. Если нужный фрагмент найден, но ответ не соответствует ему, проблема находится на этапе генерации чат-моделью.
