# Pipeline

## Требования

- Python 3.11 или новее.
- Установленный и запущенный Ollama.
- Модели `qwen2.5:3b` и `bge-m3`.

## Установка

В PowerShell из папки проекта выполните:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
ollama pull qwen2.5:3b
ollama pull bge-m3
```

## Настройка

Создайте локальный файл `.env` на основе примера:

```powershell
Copy-Item .env.example .env
```

Содержимое `.env`:

```env
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_CHAT_MODEL=qwen2.5:3b
OLLAMA_EMBEDDING_MODEL=bge-m3
```

При необходимости можно заменить URL Ollama и названия моделей. До запуска приложения должен работать сервер Ollama:

```powershell
ollama serve
```

Если Ollama уже запущен как фоновая служба, повторно выполнять эту команду не нужно.

## Запуск

```powershell
cd D:\LLMs\LLM_course_homeworks\12_pdf_to_md
.\.venv\Scripts\Activate.ps1
python -m streamlit run scripts/app.py
```

Откройте в браузере `http://localhost:8501`.

## Работа приложения

1. В боковой панели загрузите один или несколько PDF-файлов.
2. Нажмите «Сохранить и обновить индекс». Приложение сохраняет PDF в `data/documents`, создаёт рядом Markdown и извлекает изображения в папку ресурсов документа.
3. Markdown разбивается сначала по заголовкам страниц, затем на фрагменты размером до 1200 символов с перекрытием 200 символов. Для фрагментов создаются embeddings `bge-m3` и локальный индекс Chroma в `data/chroma`.
4. Задайте вопрос в чате. Приложение находит до четырёх наиболее релевантных фрагментов, передаёт их вместе с вопросом в `qwen2.5:3b` и показывает ответ с исходными фрагментами.
5. Нажмите «Очистить документы», чтобы удалить PDF, Markdown, изображения и индекс Chroma.

```text
PDF
        ↓
текстовые блоки и встроенные изображения
        ↓
Markdown + папка изображений
        ↓
разбиение по заголовкам и на фрагменты
        ↓
embedding-модель bge-m3
        ↓
локальная Chroma
        ↓
поиск до 4 релевантных фрагментов
        ↓
вопрос + контекст → qwen2.5:3b
        ↓
ответ и источники в Streamlit
```

## Ограничения

- OCR не реализован: для PDF без текстового слоя Markdown содержит соответствующее уведомление, но текст с изображения не распознаётся.
- Конвертер сохраняет текстовые блоки и встроенные изображения; сложная вёрстка, таблицы и порядок элементов могут преобразовываться неточно.
- В индекс попадают только созданные Markdown-файлы.
- Ответ зависит от найденного контекста: если нужной информации в нём нет, модель должна вернуть сообщение об отсутствии информации в загруженных документах.
