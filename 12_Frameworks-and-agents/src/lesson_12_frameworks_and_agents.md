## LangChain

Экосистема для разработки приложений на базе LLM с единым интерфейсом для моделей, векторных БД и инструментов.

### Основные возможности

- Унификация работы с различными LLM и VectorDB.
- LCEL для декларативного построения пайплайнов.
- Поддержка агентов и внешних инструментов.
- Интеграция с LangGraph и LangSmith.

### Основные модули

- **Model I/O** — работа с моделями и Tool Calling.
- **Prompts** — управление сообщениями.
- **Output Parsers** — структурированный вывод.
- **Retrieval** — RAG-пайплайны.
- **Composition (LCEL)** — построение цепочек.
- **Agentic Logic (LangGraph)** — графы и агенты.
- **Memory** — сохранение состояния.

### Retrieval (RAG)

Этапы подготовки данных:

1. Загрузка данных через Loaders.
2. Разбиение текста через Splitters.
3. Построение Embeddings.
4. Сохранение в Vector Store.
5. Поиск через Retrievers.
6. Передача найденного контекста в LLM.

### Агенты и оркестрация

- Агент работает как цикл принятия решений.
- Состояние хранится в **State Schema**.
- Поддерживаются чекпоинты и Human-in-the-Loop.
- Возможны долгоживущие диалоги и восстановление после сбоев.

### Паттерны агентов

- **ReAct**
- **Plan-and-Execute**
- **Reflection**
- **Multi-Agent**

## LlamaIndex

Фреймворк с фокусом на качество работы с данными и RAG.

### Ключевые особенности

- Data-Centric подход.
- Работа со сложными PDF, таблицами и презентациями.
- Продвинутая индексация.
- Workflow-процессы для агентов.

### Основные абстракции

- **Documents**
- **Nodes**
- **Indices**
  - VectorStoreIndex
  - SummaryIndex
  - KnowledgeGraphIndex
- **Query Engine**
- **Router**

### Дополнительные возможности

- **LlamaHub** — готовые коннекторы.
- **LlamaParse** — продвинутый парсинг PDF.
- **Router Query Engine**.
- **Recursive Retrieval**.
- **Data Agents**.

### Агентский поиск

Роутер анализирует вопрос и выбирает подходящий индекс или источник данных до выполнения поиска.

## Haystack

Инженерный фреймворк для построения RAG-систем через явные пайплайны и компоненты.

### Основные принципы

- Pipelines как направленные графы.
- Компонентный подход.
- Независимость от поставщиков моделей и БД.
- Ориентация на production-системы.

### Ключевые компоненты

- Pipelines.
- Hybrid Retrieval.
- Rankers.
- Generators.
- Document Stores.

## Ollama

Инструмент для локального запуска LLM.

### Возможности

- Локальный запуск моделей.
- Разработка без затрат на API.
- Минимальная задержка.
- Работа внутри защищённого контура.

### Modelfile

Содержит:

- Базовую модель.
- System Prompt.
- Параметры генерации.

### Управление моделями

- `ollama pull`
- `ollama create`
- `ollama run`

### Оптимизация

Поддержка GGUF и квантованных моделей.

## Итог

- LangChain предоставляет универсальную экосистему для LLM-приложений.
- LangGraph расширяет возможности агентов через графы состояний.
- LlamaIndex специализируется на работе с данными и качестве RAG.
- Haystack ориентирован на production и явное построение пайплайнов.
- Ollama позволяет запускать современные LLM локально.
- Для сложных систем важны мониторинг, трассировка, управление состоянием и агентные паттерны.
