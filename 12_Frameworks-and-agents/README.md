# RAG-тьютор по уроку 12

Это первый этап домашней работы: локальный RAG без Langfuse. Приложение отвечает только по конспекту урока «Работа с фреймворками и агентами» из `src/lesson_12_frameworks_and_agents.md` и показывает фрагменты, на которых основан ответ.

## Как устроен запрос

```text
Markdown-конспект
-> разбиение на фрагменты
-> Ollama embedding model bge-m3
-> локальная Chroma

Вопрос из браузера
-> поиск 4 похожих фрагментов в Chroma
-> вопрос + найденный контекст
-> Ollama chat model qwen2.5:3b
-> ответ и источники в Streamlit
```

Ollama запускает модели на вашем компьютере как локальный HTTP-сервис. `bge-m3` превращает текст и вопрос в векторы, чтобы Chroma могла искать смыслово похожие фрагменты. `qwen2.5:3b` получает вопрос вместе с этими фрагментами и формирует ответ. Модель не читает весь файл при каждом вопросе.

## Что выполнить самостоятельно

Создание виртуального окружения остаётся за вами. Проект не создаёт и не изменяет `.venv` автоматически.

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
Copy-Item .env.example .env
ollama pull qwen2.5:3b
ollama pull bge-m3
streamlit run app.py
```

Откройте `http://localhost:8501`, нажмите «Создать индекс» и задайте вопрос: `Какие этапы подготовки данных есть в RAG?`

## Файлы

- `app.py` — браузерный интерфейс и история сообщений в текущей сессии.
- `src/lesson_12_frameworks_and_agents.md` — копия конспекта, включённая в проект как база знаний.
- `rag_service.py` — загрузка конспекта, индексация, поиск и вызов Ollama.
- `settings.py` — читает `.env` и хранит пути/имена моделей.
- `data/chroma` — локальный индекс; он создаётся после нажатия кнопки и не попадает в Git.
