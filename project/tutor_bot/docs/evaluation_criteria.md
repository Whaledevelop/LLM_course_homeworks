# Критерии оценки Tutor Bot

## Тестовая база

Для первоначальной оценки используются 15–20 реальных Markdown-заметок.

В набор должны входить:

- русскоязычные и англоязычные заметки;
- короткие и длинные заметки;
- заметки с похожими темами;
- заметки с точными техническими терминами;
- заметки с несколькими Markdown-разделами.

## Retrieval

Минимальный набор содержит:

- 10 вопросов с одной ожидаемой заметкой;
- 5 вопросов с несколькими ожидаемыми заметками;
- 5 вопросов, на которые в базе нет ответа.

Основные метрики:

- Recall@5;
- MRR;
- средняя задержка поиска;
- p95 задержка поиска.

## Генерация ответа

Для каждого вопроса указываются:

- обязательные факты;
- допустимые источники;
- запрещённые неподтверждённые утверждения;
- ожидаемый отказ при отсутствии контекста.

Проверяются:

- полнота ответа;
- соответствие найденному контексту;
- корректность ссылок;
- отсутствие выдуманных фактов.

## Проверка заданий

Тестовый набор должен содержать:

- правильный ответ;
- частично правильный ответ;
- ответ с фактическими ошибками;
- ответ, не соответствующий условию.

Результат модели сравнивается с ручной оценкой.

## Active Recall

Для нескольких заметок подготавливаются:

- ожидаемый вопрос;
- обязательные тезисы;
- полный ответ;
- частичный ответ;
- неправильный ответ.

Проверяется, распознаёт ли система раскрытые и пропущенные тезисы.

## Начальные требования

До получения первых измерений жёсткие quality gates не устанавливаются.

Первый полный запуск фиксируется как baseline. Последующие изменения сравниваются с ним по качеству и задержке.
