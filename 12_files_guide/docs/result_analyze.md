# Анализ результатов

## Что проверяет оценка

Датасет `knowledge_base_evaluation` содержит два контрольных вопроса о загруженных документах. Для каждого вопроса сохранён ожидаемый критерий ответа: ответ должен быть основан только на содержимом документов и не добавлять отсутствующие факты.

Датасет — это набор стабильных тестовых примеров. Сам по себе он не запускает RAG и не выставляет оценки.

## LLM-as-a-Judge

После нажатия «Запустить LLM-оценку» приложение создаёт Experiment, привязанный к датасету `knowledge_base_evaluation`. Для каждого тестового вопроса Experiment:

1. Получает ответ RAG.
2. Передаёт вопрос, ожидаемый критерий и полученный ответ локальной модели `qwen2.5:3b`.
3. Записывает score `answer_groundedness` в Langfuse и связывает результат с dataset run.

Судья возвращает бинарную оценку:

- `1` — ответ удовлетворяет критерию и опирается на контекст документов.
- `0` — ответ не удовлетворяет критерию, содержит неподтверждённые сведения или не отвечает на вопрос.

Такая проверка оценивает смысл ответа, а не посимвольное совпадение с эталонной строкой. Поэтому разные корректные формулировки могут получить одинаково высокую оценку.

## Где смотреть результаты в Langfuse Cloud

1. В разделе **Datasets** откройте `knowledge_base_evaluation`, чтобы посмотреть контрольные вопросы и ожидаемые критерии.
2. В разделе **Tracing** найдите traces, созданные во время запуска оценки. В них доступны ответ RAG, найденный контекст и observation модели-судьи.
3. В разделе **Experiments** откройте созданный прогон `knowledge_base_rag_evaluation`, чтобы посмотреть ответы, оценки и сравнить dataset runs.
4. В разделе **Scores** отфильтруйте score `answer_groundedness`, чтобы сравнить результаты прогонов.

Раздел **Evaluators** может оставаться пустым. Текущая реализация не создаёт cloud evaluator в интерфейсе Langfuse: оценку выполняет локальная модель, а приложение передаёт готовый результат в Langfuse как score.

## Интерпретация

Проверку следует запускать после изменения документов, модели, prompt, параметров разбиения на фрагменты или retrieval. Сравнение оценок между прогонами позволяет обнаружить регрессии: снижение количества оценок `1` означает, что ответы стали хуже соответствовать заданным критериям.
