# Pipeline

## Выбор базы данных

Для реализации выбрана ChromaDB, потому что она запускается локально, не требует отдельного сервера и подходит для сдачи в формате GitHub-репозитория. База хранит документы, эмбеддинги и метаданные в persistent-директории `data/chroma`.

Chroma использует HNSW-индекс. Это ANN-алгоритм на графе ближайших соседей. Он дает хороший баланс скорости и качества, но требует больше памяти, чем кластерные индексы типа IVF.

## Схема данных

Каждый документ имеет поля:

- `id` - стабильный идентификатор.
- `text` - текст для индексации и генерации ответа.
- `title` - человекочитаемый заголовок.
- `category` - тема документа.
- `source` - тип источника.
- `year` - год.
- `difficulty` - уровень сложности.

Метаданные нужны для фильтрации и для отображения источников в RAG-ответе.

## Индексация

Сценарий `scripts/run_demo.py --rebuild` выполняет шаги:

1. Генерирует датасет `data/documents.jsonl`.
2. Создает persistent-клиент ChromaDB.
3. Создает коллекцию `rag_homework_cosine`.
4. Настраивает HNSW:
   - `hnsw:space=cosine`.
   - `hnsw:construction_ef=100`.
   - `hnsw:search_ef=50`.
   - `hnsw:M=16`.
5. Добавляет документы батчами по 128.

## Эмбеддинги

В проекте используется локальная hashing embedding-функция размерности 384. Она не требует внешних API и скачивания моделей, поэтому домашку можно воспроизвести в чистом окружении.

Минус такого подхода - качество ниже, чем у sentence-transformers или OpenAI embeddings. Плюс - воспроизводимость, скорость и отсутствие секретов в проекте.

## Поиск

Реализованы три режима:

- Semantic search: поиск ближайших векторов в Chroma.
- Filtered search: semantic search с `where` по метаданным.
- Hybrid search: semantic search + keyword scoring + Reciprocal Rank Fusion.

Пример фильтра:

```python
where={"category": "ann_algorithm"}
```

## RAG

RAG-сценарий:

1. Принимает вопрос пользователя.
2. Получает top-k релевантных документов.
3. Собирает контекст с title и source.
4. Формирует ответ только на основе найденного контекста.
5. Возвращает список использованных источников.

Генерация сделана extractive/template-based, чтобы проект запускался без LLM API-ключей. При необходимости этот слой можно заменить вызовом OpenAI, Hugging Face или локальной LLM, оставив retrieval без изменений.

## Оценка

Для retrieval используются:

- Recall@K.
- MRR.
- Mean latency.
- P95 latency.

Контрольные запросы находятся в `scripts/evaluation.py`. Релевантность считается по ожидаемой категории документа.
